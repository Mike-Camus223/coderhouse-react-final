# Mi proyecto final es un e-commerce llamado "Cafecito", pensada como una Single Page Application. Básicamente funciona como una tienda online donde el usuario/cliente puede recorrer un catálogo de productos, filtrarlos por categoría, ver el detalle de cada uno, agregarlos al carrito y terminar una compra. Todo esto se hace sin recargar la página, ya que la navegación se maneja con rutas internas que cambian la vista según la URL.

el e-commerce tiene distintas rutas que representan las secciones principales: la página de inicio en /, el catálogo completo en /productos, las vistas filtradas por categoría en /categoria/:categoriaId, el detalle de cada producto en /producto/:slug, el checkout en /checkout para confirmar la compra y la página final /orden/:orderId donde se muestra el ID de la orden generada. También incluye una vista para rutas inexistentes (el error 404).

Podría decirse también que la navegación está gestionada mediante React Router, el cual permite mapear cada ruta del e-commerce a un componente específico. Por ejemplo, la ruta /productos renderiza el catálogo completo, /categoria/:categoriaId permite filtrar dinámicamente según la categoría seleccionada y /producto/:slug muestra el detalle de un producto en particular. Esto hace posible mantener el estado del e-commerce en el cliente sin necesidad de recargar la página, reforzando el comportamiento de Single Page Application.

En cuanto a la estructura, el proyecto está dividido en páginas y componentes para mantener todo más ordenado y evitar tener un solo archivo gigante con toda la lógica mezclada. Las páginas representan cada vista principal (como el catálogo o el checkout), mientras que los componentes son partes reutilizables como la barra de navegación, las tarjetas de productos o el carrito. Además, hay un contexto que se encarga de manejar el estado global del carrito, y archivos separados para los datos de productos, la generación de órdenes y la conexión con Firebase.

El manejo del estado se hace con hooks de React. Se usa useState para cosas locales como formularios, estados de carga o elementos de la interfaz, y useEffect para pedir datos y actualizar lo que se muestra en pantalla cuando cambia algo, por ejemplo la categoría seleccionada o el producto que se está viendo. Esto permite que la interfaz reaccione automáticamente a los cambios.

La comunicación entre componentes se resuelve principalmente con props, pasando la información necesaria de un componente a otro. Por ejemplo, un contenedor de detalle puede enviar los datos de un producto al componente que lo muestra. Pero para el carrito se usa Context, así se evita tener que pasar datos por muchos niveles y cualquier componente puede acceder directamente a esa información.

El carrito es una parte central del e-commerce. Guarda una lista de productos con sus datos principales y la cantidad elegida, y además calcula valores como el total de items y el precio final. También expone funciones para agregar productos, eliminarlos, modificar cantidades o vaciar el carrito completo. Este estado se comparte en toda la app, por ejemplo para mostrar el total en el ícono del carrito, listar los productos en un panel lateral o generar la orden en el checkout.

Además, el uso de React Context para el carrito permite centralizar el estado global y compartirlo entre distintas vistas como /producto/:slug, /checkout y /orden/:orderId, evitando la necesidad de prop drilling y facilitando la sincronización de datos en todal e-commerce. Esto asegura que acciones como agregar productos, modificar cantidades o finalizar una compra se reflejen de forma consistente en toda la interfaz.

En cuanto a los datos, el proyecto puede trabajar tanto con información simulada como con una base real en Firebase Firestore. Por otro lado, la integración con Firebase se apoya en Firestore, que es una base de datos NoSQL en la nube. Esta permite almacenar y consultar datos de forma flexible, organizados en colecciones como products para el catálogo y orders para las órdenes generadas. A través de consultas específicas, como filtros por categoría o búsqueda por slug,l e-commerce puede renderizar la información de manera dinámica según la ruta activa, por ejemplo en /categoria/:categoriaId o /producto/:slug.

El renderizado es dinámico y también tiene casos condicionales. Por ejemplo, mientras se están cargando los datos se muestra un indicador de carga. Si un producto no tiene stock, se avisa y no se puede comprar. Si el carrito está vacío, se muestra un mensaje en lugar del listado. Y cuando se completa una compra, se redirige a la ruta /orden/:orderId, donde se muestra el ID de la orden generada.

El flujo del usuario es bastante simple: entra a la página principal en /, navega por los productos desde /productos o filtrando en /categoria/:categoriaId, puede ver el detalle de un producto en /producto/:slug, agregarlo al carrito, revisar lo que eligió y finalmente completar la compra en /checkout ingresando sus datos. Después de eso, se genera una orden y se muestra una confirmación en /orden/:orderId.
